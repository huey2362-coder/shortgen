# ShortGen — 자동 반복수익 SaaS (빌드 로드맵)

> 전략: **돈 목적 + 노력 최소.** 사용자 볼트 법칙("자동+단기+수익 중 둘")상 **단기를 버리고 자동+수익** 선택.
> ⛔ **완전 독립 트랙.** 기존 프로젝트(Bot Ledger 채널·게임·볼트)와 **연관/의존 금지**(2026-07-23 사용자 지시). 유통도 자체적으로.
> 결제 자동(구독)이라 사용자는 영업 0. 원가 ~0(무료 TTS·ffmpeg·무료 스톡). **유통 = 독립**(자체 랜딩 SEO·마켓플레이스·자체 소셜/데모).
> 엣지 = **엔진을 이미 보유**(edge-tts+자막+ffmpeg 파이프라인, 2026-07-23 검증). 남보다 빠른 출시.
> ⚠️ 정직: 몇 달 걸리고 안 터질 수 있음. **독립 유통이라 콜드스타트 = 발견성이 최대 난관**(기존 채널 이점 포기). 수익×노력최소의 최선 조합일 뿐, 보장 아님.

## 제품 한 줄
"주제 넣으면 완성된 세로 쇼츠(음성+자막) 나오는 웹 도구" — faceless 크리에이터 대상 구독제.

## 역할
- **클로드**: 빌드·운영·마케팅 콘텐츠·고객지원 초안 전부.
- **사용자(일회성)**: 도메인·호스팅·Stripe·은행 연결 → 이후 입금 + 가끔 승인. ~1%.

## 빌드 단계
- [x] **0. 제품 코어** — `core/pipeline.py` `generate_short(script)->mp4`. 검증(스모크 OK). ffmpeg/ffprobe 번들.
- [x] **1. 배경 2티어** — 무료=`background="gradient"`(애니 그래디언트+그레인+비네트, 키/GPU 0, **검증완료**) / 유료=`background="footage"`(Pexels 문장키워드별 실사 B롤 자동합성, **코드완성·키 오면 테스트**). 자막 Shadow=0으로 정리.
- [x] **2. 웹앱** — FastAPI 백엔드(`app/main.py`: /api/generate·status·download, 백그라운드 job) + 프론트(`app/static/index.html`, 다크 UI, 폴링·미리보기). **로컬 전체흐름 검증완료**(생성→다운로드). 크로스플랫폼 ffmpeg 폴백.
- [~] **3. 계정·구독** — Stripe 뼈대 완성(`app/billing.py`, env키 오면 활성). ⛔**사용자 Stripe 계정·키 필요**(테스트 불가 벽).
- [~] **4. 배포** — Dockerfile·requirements·.dockerignore 준비완료. ⛔**사용자 호스팅 계정 필요.** 절차 = `SETUP.md`.
- [ ] **5. 마케팅 (독립)** — 자체 랜딩 SEO + Product Hunt + 크리에이터 커뮤니티(Reddit/Discord) + 필요시 **제품 전용** 데모 소셜(기존 Bot Ledger와 무관). 기존 채널·브랜드에 얹지 않는다.

> ✅ **재사용 OK = 경험/기술 자산**: 영상 엔진(edge-tts+자막+ffmpeg 파이프라인)·코드·리서치 노하우는 그대로 활용(사용자 허용). ❌ **금지 = 브랜드/계정/유통 연결**(Bot Ledger 채널로 홍보·계정 공유·상호 링크).
- [ ] **6. 운영** — 사용량 모니터·고객지원 초안·개선. 스케줄 자동화.

## COGS / 가격
- 원가: TTS 무료, ffmpeg 무료, 스톡 무료, 호스팅 저가. 영상당 서버비 소액.
- 가격안: Free(워터마크/3편) → Pro $12/mo(무제한·워터마크 제거). 반복수익.

## 교훈·지뢰
- **결제 자동이 핵심** — 사람 클로징 없어야 "노력 최소"가 성립. 그래서 B2B 리테이너 아니라 셀프서브 구독.
- **비주얼 없으면 쓸모없다**(볼트 반복 교훈) — 코어는 되지만 배경화면 1장은 밋밋. 1단계(스톡 B롤)가 제품 가치의 절반.
- **플랫폼 자동화 밴 주의** — 마케팅 채널(유튜브) 과자동화 금지. 볼트 결제사/밴 교훈.
- **노력 최소 ≠ 수익 보장** — 자동화는 노력 축만 푼다. 유저가 안 사면 0. 발견성이 진짜 관문.
